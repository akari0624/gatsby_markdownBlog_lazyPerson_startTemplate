mdx的超強功能，因為mdx是markdown + JSX, 所以可以在mdx裡import 寫好的react component

  • 這裡引用joshwcomeau使用react-live做出來的 LiveEditableCode origin source
  • 注意如果在markdown之間做import，前後都要空一行，不然mdx會parse失敗，寫jsx時也必須要前後空一行
this is a text line in a div,

example: import some dependencies into scope

  • react-live的LiveProvider這個用來包在能夠"live"的程式碼片段的最外面的component有一個props叫scope，是用來讓人傳入要執行的程式碼時用到的依賴。
  • 在這個mdx檔裡先import好(btw 可以寫在最上面)，然後透過這個scope傳進被parse然後要來live的程式碼裡
  • 這邊我傳入d3-array的max, min, sum為例，下面的能夠live的code區塊就變成讓人能夠快速了解d3-array的max, min, sum是在幹麻，並且動手玩玩看的最好的範例，而且你可以在它之後用markdown語法寫更多說明
import d3ArrayMaxExample from './code/d3-array-max.example';  //載入程式碼  就是一堆字串
import LiveEditableCode from '@components/LiveEditableCode';  // 使用精心做好的 LiveEditableCode component
import { max, sum, min } from 'd3-array';

// .....  使用

<LiveEditableCode 
      id="d3-array-max"
      scope={{max, sum, min}}
      code={d3ArrayMaxExample}
      split={[66, 34]}
      maxHeight={585}/>
